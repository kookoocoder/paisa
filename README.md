# Paisa Trader

No-demat, no-broker-key research harness for Indian market experiments.

It uses:

- `yfinance` for free/delayed NSE/BSE/index candles.
- Upstox read-only tokens for broker-backed historical candles and current market quote snapshots.
- NSE public reports/bhavcopy URLs where available.
- A simulated broker for paper fills with configurable costs, slippage, latency, and spread.
- A StockSharp bridge export plus a StockSharp sample paper harness under `StockSharp/Samples/11_PaisaPaperHarness/`.
- Simple baselines so models must beat something real, not a toy benchmark.

This project does **not** place real orders. It is for research, paper trading, and building the control system before any broker account/API is connected.

## Quick Start

```bash
python3 -m venv .venv
. .venv/bin/activate
python -m pip install -e ".[dev]"

paisa symbols
paisa download --symbols RELIANCE.NS TCS.NS INFY.NS --period 60d --interval 1d
paisa download --source upstox --symbols RELIANCE.NS TCS.NS INFY.NS --period 5d --interval 5m
paisa upstox-quote --symbols RELIANCE.NS TCS.NS INFY.NS
paisa backtest --symbols RELIANCE.NS TCS.NS INFY.NS --period 60d --interval 1d --strategy ma-cross
paisa stocksharp-export --symbols RELIANCE.NS TCS.NS INFY.NS --period 60d --interval 1d --strategy ma-cross
paisa dashboard
pytest
```

Launch the live-delayed dashboard:

```bash
pip install -e ".[dashboard]"
paisa dashboard
```

Open `http://localhost:8501`, pick symbols/strategy, and click **Refresh delayed data** to pull fresh `yfinance` candles and replay the shadow paper strategy.

The dashboard gives your AI model/human operator:

- price and paper-equity charts,
- indicator overlays,
- indicator tables,
- tradability filters,
- estimated depth tables,
- next-move score/action/reasons,
- per-symbol AI snapshot JSON,
- combined AI snapshot JSON for all selected symbols,
- paper fills and current target exposure.

Depth is synthetic because there is no broker/exchange feed in the no-demat setup. It is estimated from range, volume, and spread assumptions and is clearly labeled in the UI.

Run a one-shot shadow session from the terminal (useful for cron / end-of-day reports):

```bash
paisa shadow --symbols RELIANCE.NS TCS.NS INFY.NS --period 3mo --interval 1d --strategy ma-cross --export-bridge
```

### Autonomous Intraday Web Dashboard

The custom web dashboard replays historical intraday candles as a live paper environment. The algo runs continuously with no manual refresh — open the page and watch it trade.

```bash
paisa web
```

Open `http://localhost:8080`. The engine:

- downloads delayed `yfinance` intraday candles on startup,
- streams bars in real time (1 bar/second by default),
- runs the selected strategy with an intelligence filter on entries,
- loops the session automatically when the window ends,
- pushes live updates over WebSocket (equity, fills, signals, activity log).

Useful flags:

```bash
paisa web --symbols RELIANCE.NS TCS.NS INFY.NS --interval 5m --period 5d --strategy ma-cross --tick-seconds 0.5
```

Generated data and reports are written under:

- `data/processed/`
- `reports/`

These paths are ignored by git because they are regenerated by download, backtest, shadow, and bridge-export commands.

## CLI

Show default symbols:

```bash
paisa symbols
```

Download candles:

```bash
paisa download --symbols RELIANCE.NS TCS.NS INFY.NS --period 6mo --interval 1d
```

Run a backtest:

```bash
paisa backtest --symbols RELIANCE.NS TCS.NS INFY.NS --period 6mo --interval 1d --strategy ma-cross
```

Export a StockSharp bridge package:

```bash
paisa stocksharp-export --symbols RELIANCE.NS TCS.NS INFY.NS --period 6mo --interval 1d --strategy ma-cross
```

That writes:

```text
reports/stocksharp_bridge/<strategy>_<timestamp>/
  manifest.json
  *_candles.csv
  *_signals.csv
  *_fills.csv
  *_equity.csv
  summary.csv
```

Fetch an NSE equity bhavcopy for a date:

```bash
paisa nse-bhavcopy --date 2024-06-03
```

NSE changes public-report paths over time, so bhavcopy fetch has a fallback URL strategy and will fail loudly with the attempted URLs if the exchange changes its layout.
The current implementation supports both the old `cm03JUN2024bhav.csv.zip` style archive and the newer UDiFF `BhavCopy_NSE_CM_0_0_0_YYYYMMDD_F_0000.csv.zip` style archive.

## Upstox Data

Create an Upstox Developer app, then generate an Analytics Token from the app dashboard for read-only market data. Keep the API key, API secret, and token out of git.

```bash
export UPSTOX_ANALYTICS_TOKEN="your-upstox-analytics-token"

paisa download --source upstox --symbols RELIANCE.NS TCS.NS INFY.NS --period 5d --interval 5m --force
paisa upstox-quote --symbols RELIANCE.NS TCS.NS INFY.NS
paisa upstox-quote --ltp --symbols RELIANCE.NS
```

The Upstox source accepts familiar NSE/BSE symbols such as `RELIANCE.NS` and `SBIN.BO`, then resolves them through the Upstox BOD instrument master. Raw Upstox instrument keys like `NSE_EQ|INE002A01018` are also accepted.

## Strategies

Included baselines:

- `buy-hold`: one buy at first bar, hold to the end.
- `ma-cross`: moving-average crossover with no shorting by default.
- `mean-reversion`: z-score reversion around a rolling mean.

These are intentionally simple. The point is to verify data, costs, fills, and reporting before adding ML.

## Simulated Broker

The broker models:

- cash balance,
- long-only positions,
- market order fills at next bar open,
- configurable spread/slippage basis points,
- Indian-style cost stack approximation,
- order and fill logs,
- mark-to-market equity curve.

It is not a substitute for live broker behavior. It is a controlled paper environment for development.

## StockSharp Paper Harness

The StockSharp-facing sample lives at:

```text
StockSharp/Samples/11_PaisaPaperHarness/
```

It consumes a `manifest.json` generated by `paisa stocksharp-export`, then replays the candle/signal files as a deterministic paper-trading run.

Run it with the .NET SDK:

```bash
dotnet run --project StockSharp/Samples/11_PaisaPaperHarness/11_PaisaPaperHarness.csproj \
  -- reports/stocksharp_bridge/ma-cross_YYYYMMDD_HHMMSS/manifest.json
```

This workspace has been verified with .NET SDK 10.0.301.
